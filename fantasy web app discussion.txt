# 專案背景
這是一個 Fantasy NBA Demo 專案，目前已完成的功能包含：

- 前端 H2H UI
- Yahoo API / Token 機制
- Railway Volume Token 持久化
- 全聯盟陣容頁（150+ 球員）
- FA 市場全量頁（320+ FA 球員，含搜尋、排序、推薦 badge）
- 對戰累計統計
- UI 優化（sticky nav、tab 動畫、hover、accent bar）
- 部分 7天 / 14天 / 30天 stats tab

目前待辦中的項目包含：

- 手機響應式設計
- Telegram Web App
- LINE MINI App
- 每日賽程 / 日期切換
- FA 推薦邏輯優化

# 目前正在思考的問題
我們正在重新思考產品定位：

是否真的有必要把所有功能都做進 web app？

因為目前除了 web app 之外，也可以考慮透過 Telegram 與 Discord 來提供使用者查詢、提醒、摘要與建議，所以不一定所有功能都需要做成頁面。

# 目前討論出的方向
我們目前傾向認為，不應該把所有功能都無限制地堆進 web app，原因包括：

- 前端複雜度會持續升高
- 手機版與 responsive 壓力會變大
- 相似功能可能在不同平台重複開發
- 維護成本會提高
- 產品定位會變得不清楚

因此，我們目前比較傾向讓不同平台承擔不同角色，而不是預設所有功能都必須完整存在於 web app。

# 初步平台角色想法

## Web app
比較適合承載以下類型功能：
- 結構化資訊瀏覽
- 視覺化展示
- 多球員比較
- 搜尋、排序、篩選
- dashboard 式決策介面
- tab / table / card 型資訊呈現

初步定位為：
**核心資料瀏覽與深度分析介面**

## Telegram / Discord
比較適合承載以下類型功能：
- 快速問答
- 對話式查詢
- 即時提醒
- 摘要型資訊輸出
- 個人化建議
- 自然語言資訊需求

初步定位為：
**高頻、快速、互動式的資訊入口**

# 目前對功能分工的想法

## 比較適合留在 web app 的功能
- 全聯盟陣容頁
- FA 市場全量頁
- 搜尋 / 排序 / 推薦 badge
- 7天 / 14天 / 30天統計 tabs
- 對戰 dashboard 類頁面
- UI / UX 優化
- 手機版優化
- 需要表格化、視覺化、可比較的球員資料頁

## 比較適合由 Telegram / Discord 處理的功能
- 今天誰有比賽
- 本週對戰摘要
- 哪些 categories 落後
- FA 建議
- stream 建議
- 球員 injury / news / 異動提醒
- 自然語言查詢

## 可能適合雙平台共存，但角色不同的功能
- FA 推薦
- 對戰摘要
- 球員建議
- 賽程 / matchup 資訊

可能的分工方式是：
- web app 提供完整、可瀏覽、可比較的頁面
- Telegram / Discord 提供摘要、問答、提醒或導流

# 目前的顧慮
接下來如果同時做：
- web app 持續擴功能
- Telegram Web App
- Discord 互動
- LINE MINI App

可能會導致：
- 產品面過寬
- 優先順序混亂
- 開發資源被分散
- 維護成本提高
- 功能重複

因此，我們希望先重新定義各平台邊界，再決定下一步的產品重點。
